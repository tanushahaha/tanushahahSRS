Course Registration System SRS

Introduction

This document outlines the Software Requirements Specification (SRS) for the Course Registration System.

Purpose

The purpose of this document is to provide a detailed description of the functionalities, system models, and requirements for the Course Registration System. This system is designed to help students register for courses, view available courses, and manage their schedules. Additionally, it provides functionalities for administrators to manage course offerings, student records, and registration timelines.

Scope

The Course Registration System will allow students to browse and register for courses. Administrators will be able to manage course offerings, including adding or removing courses, and setting registration timelines. The system will ensure that students can only register for courses for which they meet the prerequisites and that courses do not conflict with each other in the schedule.

Definitions, Acronyms, and Abbreviations

  • SRS - Software Requirements Specification
  • CRUD - Create, Read, Update, Delete
  • GUI - Graphical User Interface
  • DBMS - Database Management System

System Overview

The Course Registration System will be a web-based application with a user-friendly interface. The system will consist of a frontend for the users (students and administrators) to interact with and a backend to handle all the data processing, business logic, and database interactions. The system will be developed using a combination of HTML, CSS, and JavaScript for the frontend, and a suitable backend technology such as Flask or Django for server-side processing.

Functional Requirements

  • Students should be able to view available courses.
  • Students should be able to register for courses.
  • Students should be able to view their current schedule.
  • Administrators should be able to add or remove courses.
  • Administrators should be able to manage student records.

Non-Functional Requirements

  • The system should be secure and protect student data.
  • The system should be scalable to handle a large number of students.
  • The system should have a user-friendly interface.
  • The system should be available 24/7 with minimal downtime.
  • The system should be responsive and accessible on different devices.

System Models

        Student {

        Course {

Use Cases

Use Case 1: Register for a Course

Actors: Student

Preconditions: Student is logged in.

Scenario: The student selects a course and confirms registration. The system checks for conflicts and adds the course to the student's schedule.

Postconditions: The course is added to the student's schedule, and the course seat count is updated.

Use Case 2: Add a Course Offering

Actors: Administrator

Preconditions: Administrator is logged in.

Scenario: The administrator adds a new course offering with details such as course name, schedule, and instructor. The system saves the course offering to the database.

Postconditions: The new course offering is available for students to register.

Conclusion

The Course Registration System is designed to streamline the process of course registration for students and administrators, providing a secure, scalable, and user-friendly interface. The system will facilitate efficient management of course offerings and student schedules, ensuring that students have access to the courses they need to progress in their academic journey.
